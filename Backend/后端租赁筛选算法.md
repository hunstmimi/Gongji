# 后端租赁筛选算法

## 当前模型

- 用户按卡租用。
- 请求参数为：`card_type`、`cabinet_type`、`card_count`。
- 后端根据系统时间自动判断时段：
  - `09:00` 到 `17:00` 为白天 `day`
  - 其他时间为夜间 `night`

## 机柜状态规则

- `offline`：机柜当前没有任何卡在跑，`active_card_count = 0`
- `available`：机柜里有卡在跑，但还没跑满，`0 < active_card_count < capacity_cards`
- `rented`：机柜卡位已满，`active_card_count = capacity_cards`

只要一个机柜里没有任何卡在跑，就立即关机并记为 `offline`。

## 筛选步骤

1. 后端先按 `card_type` 和 `cabinet_type` 过滤机柜。
2. 只保留还能继续分配卡位的机柜：
   - `status in ('available', 'offline')`
   - `active_card_count < capacity_cards`
3. 根据当前时段计算每张卡的电费：
   - 白天：`day_hourly_power_cost / capacity_cards`
   - 夜间：`night_hourly_power_cost / capacity_cards`
4. 按下面顺序排序：
   - 每张卡电费从低到高
   - 同价时优先已经开机的 `available`
   - 再同价时优先当前负载更高的机柜
   - 最后按 `cabinet_code` 升序稳定排序
5. 从排序后的机柜开始，按剩余卡位逐个分配，直到凑满用户要的 `card_count`。
6. 如果全部可分配卡位之和仍小于请求卡数，返回 `NO_AVAILABLE_CARDS`。

## 价格规则

- 用户价格按卡数阶梯计算，和电费分开。
- 电费始终按“每张卡耗电相同”处理。
- 每个机柜的电费总额 = `每张卡电费 * 本机柜本次分配卡数`
- 整个租单的电费总额 = 所有分配机柜电费之和

当前内置价格规则：

- `4090 / 单卡机柜`：`12.0 * card_count`
- `910C / 16卡机柜`：`1卡 4.0`，`2卡及以上 3.9 * card_count`

## 分配后的状态更新

- 某机柜被分到 `n` 张卡后：
  - `active_card_count += n`
  - 然后重新计算状态
- 租单取消后：
  - 对每个分配机柜执行 `active_card_count -= allocated_cards`
  - 如果减完为 `0`，立即变成 `offline`
  - 如果还有空余卡位，变成 `available`
  - 如果仍满载，保持 `rented`
